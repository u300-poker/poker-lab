# PokerLab (포커랩) - 포커 자가교육 프로그램

## 문제
포커를 두는 사람들이 티켓을 따내기 위해, 집에서 게임을 하는경우가 많다.

그리고 실력 향상을 위해서는 포커 한 판이 끝났을 때, 복기를 통해 본인이 실수한 것이 무엇인지, 왜 실수했는지를 알아야한다. 하지만 현재 그럴만한 수단이 마땅히 없다.

## 가설
포커를 두는 사람들은 실력 향상을 위해, 자신의 포커를 복기한다.
복기하는 과정이 불편하다.
복기하는 과정을 도와주면 돈을 낼 것이다.

# PokerLab (포커랩) MVP 기획

## 1. 개요 및 목적
- **서비스명:** PokerLab (포커랩)
- **슬로건:** 당신의 포커 실력을 연구하고 교정하는 AI 퍼스널 코치
- **목적:** 온라인 포커(WPL 등) 유저들이 세션 녹화본을 통해 자신의 플레이를 과학적으로 복기하고, 실전 실력을 향상시킬 수 있도록 돕는 코칭 프로그램.
- **핵심 가치:** 
    - **Post-Game Analysis:** 실시간 부정행위(RTA) 방지를 위해 철저히 사후 복기에 집중.
    - **Visual Sync Coaching:** 플레이 영상과 AI 피드백, EV 비교 차트를 한 화면에서 직관적으로 대조.
    - **Learning Loop:** 유저의 약점 데이터를 누적하여 맞춤형 복습 퀴즈 제공.

## 3. 플랫폼 및 수익 모델 (BM)
- **플랫폼:** Web 서비스 (접근성 우선, 데스크탑/모바일 모두 브라우저를 통해 사용 가능)
- **수익 모델:** 종량제 (Pay-as-you-go)
    - 예: 크레딧 충전 방식. 1회 핸드 이미지 분석 및 코칭 피드백 생성 당 일정 크레딧 차감.

## 4. 데이터 플로우 및 핵심 기능 (MVP)

### Step 1: 데이터 입력 (OCR 기반)
- 유저가 WPL 등에서 게임 플레이 후 저장/캡처한 '핸드 히스토리 요약 이미지'를 웹 서비스에 업로드 (또는 클립보드 붙여넣기).
- **Vision AI / OCR 기술**을 사용하여 이미지에서 텍스트(플레이어 포지션, 스택, 블라인드, 액션, 보드 카드, 히어로의 핸드 등)를 추출하고 정형화된 JSON 형태의 데이터로 변환.
- **데이터 정책:** 추출 완료 후 원본 이미지는 익명화하여 우리 서버의 추가 학습(인식률 개선 및 상황 데이터 누적) 용도로 보관.

### Step 2: 코칭 및 피드백 (핵심)
- 추출된 정형 데이터를 바탕으로 AI (GPT-4o 등) 및 GTO 기반 로직이 해당 핸드의 액션을 분석.
- **UI 형태:** 1차적으로는 특정 핸드에 대한 AI의 상세한 텍스트 코칭(챗봇/게시판 형태)을 제공.
    - 예: "플랍에서 체크-레이즈를 한 것은 좋은 시도였으나, 상대가 UTG에서 오픈했고 보드가 A-K-2로 상대 레인지에 유리하기 때문에 빈도를 줄이거나 콜만 하는 것이 좋습니다."

### Step 3: 데이터 누적 및 트레이닝 (장기 목표 기반)
- **MVP 핵심 집중 분야:** 분석된 핸드 데이터(상황, 유저의 선택, 정답/오답 여부, AI 피드백)를 유저별 DB에 체계적으로 누적.
- 누적된 데이터를 바탕으로, 추후 유저가 자주 틀리는 상황(예: 리버에서의 블러프 캐치, 특정 포지션에서의 3-Bet 방어 등)을 분석.
- 해당 약점 상황들을 **'퀴즈 형태'**로 재생성하여 유저가 반복 학습(트레이닝)할 수 있는 시스템 구축을 MVP 단계부터 염두에 두고 DB 스키마 설계.
